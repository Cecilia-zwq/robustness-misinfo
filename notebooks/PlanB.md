# Wrapper: make the user agent and the target model into different scripts/functions so that it’s easy to toggle them and change the personas

The evaluation framework has three main components: simulated user agent, target LLM, and evaluator.

- The user agent is simulated by LLM. It contains an actor module and a character reflection module. The actor module is used for setting up user types, for example, demographic vs conservative, emotionally vs rational, and kind vs aggressive. The user types are the independent variables in the future experiment to see under what scenario, the model is most vulnerable. Therefore, actor module can output message to target LLM and recive the reponse and then reply it. For the first message, the actor module needs to generate an opening message for its false claim. Later turns, it will be asked to reply to the target LLM's response. For the actor module to be consistent in its character or persona, the character reflection module is used for audit to ensure that the message generated by the actor module does not break its role. If the message breaks the character and fails to stand up for the false claim, the actor module will be asked to regenerate its message. If the message does not break character and keep standing for misinformation, the message will be passed to the target LLM.

- Target LLM is the model that is under evaluation. By default, it does not have any API setting. After receiving the user message, LLM will generate its response. The response will be passed to the user agent for the next round of conversation and to the evaluator to score the LLM.

- The evaluator is the metric for the whole experiment. It is meant to measure LLMs' robustness against misinformation.
